    client.config

    The clients configuration (Most from Discordia client)

    Fields Type Optional Default Description
    prefix string ☑️ '!' The prefix the bot will use
    defaultHelp boolean ☑️ true Whether to use the default help or not
    owners table ☑️ {} The owners of this bot
    testing boolean ☑️ false Weather the bot is in testing or not
    testbot boolean ☑️ false Weather this current bot is a test bot
    botid string ☑️ nil The bot testing this current bot
    storeErrors boolean ☑️ false Weather to store error information within a table
    disableDefaultCH boolean ☑️ false Weather to disable the default command handler (Usually for custom prefixes)
    routeDelay number ☑️ 300 Minimum time in milliseconds to wait between HTTP requests per-route
    maxRetries number ☑️ 5 The maximum number of retries to attempt after an HTTP request fails
    shardCount number ☑️ 0 The total number of shards the application is using (0 signals to use the recommended count)
    firstShard number ☑️ 0 The ID of the first shard to run on the client (0 is the minimum)
    lastShard number ☑️ -1 The ID of the last shard to run on the client (-1 signals to use shardCount - 1)
    largeThreshold number ☑️ 100 Limit to how many members are initially fetched per-guild on start-up
    cacheAllMembers boolean ☑️ false Whether to cache all members (If false, offline members may not be cached)
    autoReconnect boolean ☑️ true Whether to attempt to reconnect after an unexpected gateway disconnection
    compress boolean ☑️ true Whether to allow for compressed gateway payloads
    bitrate number ☑️ 64000 The default bitrate to use for voice connections, from 8000 to 128000
    logFile string ☑️ 'discordia.log' The file to use for logging
    logLevel string ☑️ logLevel.info The level to use for logging (see Enumerations)
    dateTime string ☑️ '%F %T' The date and time format to use logging
    syncGuilds boolean ☑️ false Whether to automatically sync all guilds on start up (user-accounts only)
